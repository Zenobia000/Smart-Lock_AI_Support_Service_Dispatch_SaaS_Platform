# 進度報告：個人化記憶機制與 Slots 預填 (2026-03-07)

## 📌 已完成項目

承接 `docs/20260306-進度報告.md` 的架構，本次完成會議決議第 5 項「個人化記憶機制：動態使用者輪廓」的完整實作，並在此基礎上新增 Slots 預填功能與轉接真人自動帶入個人資訊。

### 1. 動態使用者輪廓 (User Profile) — 載入與更新

- **問題描述：** 系統缺乏跨 Session 的長期使用者記憶，每次對話都像面對全新使用者，無法提供個人化服務。
- **實作方式：**
    - **`graph/nodes.py`**：新增 `load_user_profile` 與 `update_user_profile` 兩個節點。
        - `load_user_profile`：在對話開始時（`START` 之後第一個節點）從 `.md` 檔案載入使用者輪廓，注入 `GraphState.user_profile`。
        - `update_user_profile`：在對話結束前（`generate`、`out_of_domain`、`human`、`ask_missing_slots` 之後），由 LLM 從本輪對話中萃取個人資訊（品牌型號、居住環境、地址、電話等），合併至既有輪廓並存檔。
    - **`graph/state.py`**：新增 `user_profile: str` 欄位。
    - **`graph/builder.py`**：將 `load_user_profile` 接在 `START` 之後、`rewrite_query` 之前；將 `update_user_profile` 接在所有終端節點與 `END` 之間。
    - **`graph/nodes.py` (`rewrite_query`)**：改寫邏輯納入 `user_profile`，讓問題改寫能利用使用者背景補充品牌型號等資訊。
    - **`graph/nodes.py` (`generate_answer`)**：回覆生成時納入使用者輪廓，提供個人化回覆。
    - **`profiles/`**：新增 `ProfileManager` 模組，負責 `.md` 輪廓檔案的讀寫。
- **影響範圍：** `graph/nodes.py`、`graph/state.py`、`graph/builder.py`、`profiles/`
- **驗證劇本：**
    - 劇本 6：首次建立輪廓（使用者提及 Samsung SHP-DP609 + 公寓大樓）
    - 劇本 7：跨 Session 記憶驗證（不提品牌型號，系統仍能根據輪廓回答）

### 2. 轉接真人自動帶入個人資訊

- **問題描述：** 轉接真人時，系統要求使用者填寫所有聯絡資訊（地址、電話、設備品牌型號），即使這些資訊已在過往對話中提供過。
- **實作方式：**
    - **`graph/nodes.py` (`transfer_to_human`)**：從 `slots` 取得品牌型號，從 `user_profile` 用 regex 提取電話與地址，自動帶入轉接表單。已有資訊的欄位顯示「確認並補充」，全空則顯示「留下以下資訊」。
- **影響範圍：** `graph/nodes.py`
- **驗證劇本：**
    - 劇本 8：第一輪提供地址電話，第二輪轉接時自動帶入

### 3. 從使用者輪廓預填 Slots

- **問題描述：** 即使輪廓中已有品牌型號（如 Samsung SHP-DP609），新 Session 的 `slots` 仍為空，導致需要 slot filling 的意圖（如 troubleshooting）會再次追問已知資訊。
- **實作方式：**
    - **`graph/nodes.py` (`load_user_profile`)**：載入輪廓後，若 `REQUIRED_SLOTS` 存在，透過 LLM 從輪廓文字中提取對應欄位值，預填至 `slots`。
    - 提取結果僅保留 key 存在於 `REQUIRED_SLOTS` 中且值非 null 的欄位，防止 LLM 回傳不合法的 key。
    - **衝突處理（免費得到）**：`extract_slots` 的合併邏輯（`{**current_slots}` + LLM 回傳非 null 才覆蓋）天然處理衝突 — 使用者當前對話中提供的值優先，未提及的欄位保留輪廓預填值。
- **影響範圍：** `graph/nodes.py`
- **驗證劇本：**
    - 劇本 9：用 demo_6 既有輪廓，發送 troubleshooting 問題但不提品牌型號，系統應從輪廓預填 slots 並跳過 `ask_missing_slots`
