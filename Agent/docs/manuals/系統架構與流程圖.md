### 2. 轉接真人策略

Agent 遵循「自主解決優先」原則，資訊不足時的處理順序：

1. 先提供能找到的部分資訊
2. 嘗試用不同關鍵字或角度重新檢索
3. 根據專業知識提供通用建議或常見做法
4. 建議使用者提供更具體的資訊
5. 只有當使用者明確表示以上方式都無法滿足需求時，才詢問是否需要轉接真人客服

`transfer_to_human` 工具僅在以下情況使用：
- 使用者明確堅持要求轉接真人客服
- 涉及安全風險（門鎖無法上鎖、疑似被破壞）

### 4. 時序圖 — 單一意圖場景 (Sequence Diagram)

```mermaid
sequenceDiagram
    actor User as LINE 使用者
    participant App as app.py<br/>(防抖層)
    participant PP as pre_process
    participant MM as manage_memory
    participant RT as router
    participant Agent as Agent 子圖<br/>(e.g. product_expert)
    participant Tool as Tool<br/>(e.g. db_smartlock_manual)
    participant MA as merge_answers
    participant UP as update_profile
    participant Post as post_process
    participant LLM as LLM Engine
    participant Profile as User Profiles<br/>(./data/profiles/)
    participant SQLite as SQLite<br/>(data/db/chat_history.db)

    User->>App: 傳送 LINE 訊息
    App->>App: 審計日誌：記錄 user_raw
    App->>App: 訊息緩衝 (buffer_wait)
    App->>App: 審計日誌：記錄 user (合併後)
    App->>PP: 合併後的 question

    PP->>Profile: 載入 user profile
    Profile-->>PP: 輪廓文字 (or empty)
    PP-->>MM: messages + user_profile + summary

    alt messages > 閾值 (6)
        MM->>LLM: 語意摘要壓縮 (summarize_messages.md)
        LLM-->>MM: 結構化摘要
        MM->>MM: RemoveMessage 刪除舊訊息，保留最近 2 對
    else messages <= 閾值
        MM->>MM: 跳過壓縮
    end
    MM-->>RT: state (含 summary)

    RT->>LLM: 意圖分類 (router.md prompt)
    LLM-->>RT: intent = "general_knowledge"
    RT->>RT: 狀態清洗 (只保留 summary + 當前問題)
    RT-->>Agent: Send(product_expert, clean_state)

    loop agent_llm ↔ tool_node
        Agent->>LLM: system prompt + messages
        LLM-->>Agent: tool_call (db_smartlock_manual)
        Agent->>Tool: 執行檢索
        Tool-->>Agent: 檢索結果
        Agent->>LLM: tool result
        LLM-->>Agent: 最終回覆 (文字)
    end

    Agent-->>MA: state (含 AI messages)
    MA->>MA: 提取 answer + 偵測 topic_resolved

    MA-->>UP: answer
    UP->>LLM: update_profile.md prompt
    LLM-->>UP: 更新後的輪廓
    UP->>Profile: 儲存輪廓

    UP-->>Post: state
    Post->>Post: Markdown 清洗 + 記錄 chat_history
    Post-->>App: answer
    App->>App: 審計日誌：記錄 ai
    App->>User: Reply / Push API
```

### 5. 時序圖 — 多意圖平行場景 (Sequence Diagram)

```mermaid

    participant MM as manage_memory

    Profile-->>PP: 輪廓文字

    MM->>MM: 檢查是否需要壓縮記憶

    RT->>LLM: 意圖分類

    par Send() fan-out 平行派發
        RT->>PE: Send(product_expert, clean_state)
        RT->>TS: Send(troubleshooter, clean_state)
    end

    par Agent 平行執行
        PE->>LLM: product_expert prompt
        LLM-->>PE: tool_call

        Tool_PE-->>PE: 結果

    and
        TS->>LLM: troubleshooter prompt
        LLM-->>TS: tool_call
        TS->>Tool_TS: 檢索排除步驟
        Tool_TS-->>TS: 結果
        TS->>LLM: tool result
        LLM-->>TS: 回覆 B (排除步驟)
    end

    PE-->>MA: 回覆 A
    TS-->>MA: 回覆 B

    MA->>LLM: merge_answers.md prompt + 回覆 A + 回覆 B
    LLM-->>MA: 合併後的統一回覆

    MA-->>UP: 合併 answer
    UP->>LLM: update_profile.md prompt
    LLM-->>UP: 更新後的輪廓
    UP->>Profile: 儲存輪廓

    UP-->>Post: state
    Post->>Post: Markdown 清洗 + 記錄 chat_history
    Post-->>App: answer
    App->>User: Reply / Push API
```

### 6. 時序圖 — 轉接真人場景 (Sequence Diagram)

```mermaid
sequenceDiagram
    actor User as LINE 使用者
    participant App as app.py<br/>(防抖層)
    participant PP as pre_process
    participant MM as manage_memory
    participant RT as router
    participant Agent as Agent 子圖<br/>(e.g. troubleshooter)
    participant Tool as transfer_to_human
    participant MA as merge_answers
    participant UP as update_profile
    participant Post as post_process
    participant LLM as LLM Engine
    participant Profile as User Profiles

    User->>App: "門鎖打不開，我要找真人客服！"
    App->>PP: question

    PP->>Profile: 載入 user profile
    Profile-->>PP: 輪廓 (含地址、電話等)
    PP-->>MM: messages + user_profile

    MM->>MM: 檢查是否需要壓縮記憶
    MM-->>RT: state

    RT->>LLM: 意圖分類
    LLM-->>RT: intent = "troubleshooting"
    RT->>RT: 狀態清洗
    RT-->>Agent: Send(troubleshooter, clean_state)

    Agent->>LLM: troubleshooter prompt + messages
    LLM-->>Agent: tool_call (transfer_to_human)
    Agent->>Tool: 執行轉接
    Tool-->>Agent: 格式化轉接文字 (含已知個資)
    Agent->>LLM: tool result
    LLM-->>Agent: 轉接回覆

    Agent-->>MA: state
    MA->>MA: 偵測 transfer_to_human → topic_resolved
    MA->>MA: 使用 tool 回傳的格式化文字作為 answer

    MA-->>UP: answer + topic_resolved
    UP->>LLM: update_profile.md prompt
    LLM-->>UP: 更新後的輪廓
    UP->>Profile: 儲存輪廓

    UP-->>Post: state
    Post->>Post: Markdown 清洗 + 記錄 chat_history + topic_resolved
    Post-->>App: answer
    App->>User: 轉接訊息 (含個資確認表單)
```
